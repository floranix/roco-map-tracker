# 2D Game Map Localizer

> 这份文档主要保留早期定位方案与技术背景，当前项目状态、GUI 功能和 biliwiki 采集路线能力请优先查看仓库根目录的 `README.md`。

一个基于图像识别的 2D 游戏地图实时定位项目。

目标是：给定一张完整游戏地图，以及用户当前所在区域的实时 2D 俯视画面，持续识别用户在大地图中的位置，并在用户移动过程中保持稳定跟踪。

## 项目背景

本项目适用于以下场景：

- 已知一张完整的 2D 俯视地图
- 实时输入是玩家当前所在区域的局部 2D 俯视画面
- 玩家会不断移动
- 希望实时输出玩家在大地图上的位置坐标
- 希望方案可先快速验证，再逐步演进到高性能版本

本项目优先采用传统计算机视觉方案，而不是目标检测或大模型方案。原因是这类问题本质上更接近图像配准与视觉定位，传统 CV 在延迟、可控性、部署成本和工程可解释性上更有优势。

## 核心思路

整体采用“两阶段定位”：

1. 全局定位
   - 在首次进入、丢失跟踪或发生大幅跳转时，从完整地图中重新找到当前区域位置。

2. 局部跟踪
   - 当上一帧位置已知时，只在附近区域内继续搜索，减少全图匹配带来的计算开销。

为了提高实时性与稳定性，方案会结合以下机制：

- 特征点匹配
- 几何验证
- 帧间运动跟踪
- 状态平滑
- 低置信度重定位

## 推荐技术方案

推荐使用：

- Python 3.10+
- OpenCV
- NumPy
- PyCharm 进行开发和调试

这一阶段目标是快速验证算法是否适合具体游戏地图，而不是先做高性能优化。

### 第二阶段：工程化与性能优化

推荐使用：

- C++17
- OpenCV
- CMake
- CLion 进行 C++ 版本开发

这一阶段用于将核心路径迁移到 C++，降低延迟并提升吞吐。

## 算法选型

### 推荐默认方案

- 特征提取：`ORB` 或 `AKAZE`
- 特征匹配：`BFMatcher` 或 `FLANN`
- 错误剔除：`RANSAC`
- 位置估计：`Homography`
- 连续跟踪：`Lucas-Kanade Optical Flow`
- 状态平滑：`Kalman Filter`

### 选择理由

- 对局部遮挡有一定鲁棒性
- 对轻微旋转、缩放和视角扰动比模板匹配更稳
- 可以较好支持玩家持续移动
- 工程实现简单，便于调试
- 后续可平滑升级到更强的深度特征匹配方案

### 为什么不优先使用目标检测或大模型

- 目标检测更适合识别“对象”，不擅长做高精度位置配准
- 大模型推理成本高，延迟大，不适合高频实时定位
- 该问题本质是“局部图像在大图中的匹配与跟踪”，传统 CV 更直接

## 系统架构

系统可以拆分为以下模块：

### 1. 地图预处理模块

负责对完整地图进行离线处理，包括：

- 灰度化
- 多尺度金字塔构建
- 地图切块
- 特征缓存
- 固定 UI 或无效区域掩码处理

### 2. 全局定位模块

用于首次定位或跟踪失败后的重定位，包括：

- 对输入局部图提取特征
- 在完整地图或地图切块中寻找候选区域
- 进行特征匹配
- 通过 RANSAC 过滤误匹配
- 输出局部图在大地图中的位置

### 3. 局部跟踪模块

用于连续帧追踪，包括：

- 根据上一帧位置确定局部搜索窗口
- 在局部区域中继续匹配
- 估计玩家移动方向和位移
- 在小范围内快速更新位置

### 4. 时序平滑模块

用于提升轨迹稳定性，包括：

- 卡尔曼滤波
- 光流辅助
- 置信度评分
- 漂移检测与回退重定位

## 典型工作流程

### 首次定位

1. 读取完整地图
2. 对完整地图做预处理
3. 读取当前局部画面
4. 提取特征点和描述子
5. 在地图中执行全局匹配
6. 使用 RANSAC 验证几何一致性
7. 输出当前位置坐标

### 连续定位

1. 以上一帧位置为中心建立局部搜索区域
2. 对当前帧局部图像进行特征匹配
3. 结合光流估计帧间位移
4. 用卡尔曼滤波平滑位置变化
5. 若置信度下降，则触发全局重定位

## Python 原型开发

适合使用 PyCharm。

### 推荐依赖

```txt
opencv-python
numpy
PyYAML
```

### 安装方式

```bash
pip install -r requirements.txt
```

### 启动方式

```bash
python main.py --gui
```

如果仓库中存在 `configs/rocom_17173.yaml`，程序会默认优先加载这份配置，把 `17173` 的整张世界地图作为完整地图使用。

在 macOS 上也可以直接双击根目录下的 `launch_gui.command` 启动图形界面。

GUI 现已支持把输入源切换到“屏幕区域”，先框选一个矩形区域，再以较短时间间隔持续抓取该区域并实时定位。

如果需要命令行模式，可以使用：

```bash
python main.py --frame data/samples/frame.png --map data/full_map.png --save-visualizations
```

如果想直接做持续屏幕采集，可以使用：

```bash
python main.py --config configs/rocom_17173.yaml --screen-region 1800,260,220,220 --capture-interval-ms 150 --visualize
```

如果要直接使用已经接入好的洛克王国世界整图与点位数据，可以运行：

```bash
python main.py --config configs/rocom_17173.yaml --gui
```

## 地图与点位的使用方式

这里的“地图输入”是整张完整大地图，不是一小块区域。当前程序默认把 `17173` 的世界地图当作整图，玩家截图、录屏或视频帧只是“当前所在区域的局部视野”。

点位标注主要用于后续路线规划，所以默认不常驻铺满整张图，而是按需展示：

- 勾选“显示点位”后，仍需要在右侧分类列表里选择至少一个分类，才会真正叠加点位
- 可以配合“关键词”进一步筛选点位名称
- “显示名称”默认关闭，避免整图文字过密
- 未选择分类时，程序不会在地图上绘制任何点位
- 当分类或点位记录里带有网站 icon 时，会优先按 icon 绘制；只有缺 icon 时才回退到圆点标记

这样可以把整图继续作为定位底图使用，同时把点位图层当作可开关的规划参考层。

当前这套接入不再依赖封面图的默认缩放，而是直接使用 `17173` 网页最大缩放层的瓦片进行拼接，因此更适合与高缩放、细节更具体的局部输入画面做匹配。

## 配置项建议

示例配置文件：

```yaml
map_path: data/full_map.png
feature_type: orb
min_match_count: 12
roi_expand_pixels: 120
resize_ratio: 1.0
use_optical_flow: true
use_kalman: true
max_lost_frames: 10
```

配置项说明：

- `map_path`：完整地图路径
- `feature_type`：特征算法类型，推荐 `orb` 或 `akaze`
- `min_match_count`：最少有效匹配点数
- `roi_expand_pixels`：局部搜索区域扩展像素
- `resize_ratio`：图像缩放比例
- `use_optical_flow`：是否启用光流辅助跟踪
- `use_kalman`：是否启用卡尔曼滤波
- `max_lost_frames`：连续丢失多少帧后触发重定位

## 输入与输出

### 输入

- 一张完整地图
- 一组局部截图，或实时视频帧流

对于当前 `17173` 接入版本，建议按下面的方式理解输入：

- 完整地图：整张世界地图
- 局部输入：玩家当前所在区域的截图、连续截图目录，或录屏视频

### 输出

建议输出以下内容：

- 地图坐标 `x`
- 地图坐标 `y`
- 朝向角 `theta`，如果可以估计
- 当前定位置信度 `score`
- 当前状态：
  - `跟踪中`
  - `重定位中`
  - `已丢失`

## 性能优化建议

- 不要每一帧都做全图匹配
- 已知上一帧位置时，只做局部搜索
- 离线缓存地图特征，避免重复计算
- 对输入帧进行合理降采样
- 使用地图切块减少候选区域数量
- 使用多线程拆分采集、匹配、绘制和输出
- Python 版本用于验证，C++ 版本用于生产部署

## 何时需要升级到更强方案

当出现以下问题时，可以考虑升级到深度特征匹配：

- 地图中存在大量重复纹理
- ORB/AKAZE 的区分能力不足
- 缩放和旋转变化较大
- 局部遮挡严重
- 误匹配较多，重定位不稳定

升级方向可以是：

- `ALIKED / SuperPoint + LightGlue`
- `ONNX Runtime`
- `TensorRT`

但建议在传统 CV 方案确认不足之后再做升级，不建议一开始就增加系统复杂度。

## 当前阶段的最小可行目标

建议先完成一个最小可行版本：

- 输入一张完整地图
- 输入一张局部截图
- 输出该局部截图在完整地图中的位置
- 支持连续帧位置更新
- 支持简单可视化结果

只要这个版本稳定跑通，再逐步增加：

- 光流辅助
- 卡尔曼平滑
- 自动重定位
- C++ 迁移
- 性能压测

如果当前手头已有 PyCharm 和 CLion，建议采用以下路线：

- PyCharm 负责快速验证 Python 原型
