# roco-map-tracker

这是当前项目的中文交接 README，目标是让下一位继续接手的人能快速知道：

- 现在已经做到什么程度
- 当前方案为什么能用
- 还有哪些明显短板
- 下一步最值得继续做什么

## 项目目标

项目目标是基于《洛克王国世界》的世界地图与玩家小地图输入，在桌面端持续识别玩家所处位置，并把位置、点位和资源路线一起显示在可缩放的大地图上。

当前工程里，这件事已经不只是“离线单图实验”，而是有 GUI、持续采集、实时更新、点位筛选和资源路线的可运行原型。

## 当前阶段

当前处于：

`可用于常见路线跟随场景，但全局重定位仍需继续优化`。

更具体一点：

- 如果用户沿着资源路线正常移动，当前已经可以做到持续定位，并在地图上更新位置。
- 对“每 1-2 秒定位一次并展示位置”这个目标，当前阶段已经能满足。
- 真正的瓶颈主要不在连续跟踪，而在首次启动后的冷启动重定位、丢失后的全局重定位，以及纯色/弱纹理/遮挡场景。

## 当前性能结论

下面是 2026-04-13 在本地样例上的直接实测结论。

### 连续跟踪阶段

在同一区域已经锁定、持续跟踪时：

- `20260413-114459.jpg`：约 `32-36ms/帧`
- `20260413-135037.jpg`：约 `172-177ms/帧`

这意味着：

- 常见路线跟随场景下，`1-2 秒定位一次` 没有问题
- 很多场景下甚至可以更快
- 当前默认屏幕采集间隔是 `250ms`

### 冷启动 / 全局重定位阶段

冷启动时：

- `LocalizationPipeline` 初始化大约 `2.1s-2.3s`
- 首帧全局识别大约 `2.8s-3.0s`

所以：

- 第一次启动或彻底丢失后重新锁定，体感仍然偏慢
- 真正影响“实时感”的主要是重定位，不是稳态跟踪

## 现在能不能用于“沿资源路线走图”

可以，但要带着边界理解。

当前已经能支持这样的常见用法：

1. 打开 GUI
2. 选择一块小地图屏幕区域
3. 选择资源来源和资源路线
4. 勾选“启用识别”
5. 玩家沿着资源路线移动时，地图上持续更新当前位置

对这类连续移动场景，当前阶段已经是“可用原型”。

不应夸大的地方也要说明：

- 不是所有场景都能稳定高频更新
- 丢失后重新定位仍然慢
- 纯色区域、弱纹理区域、严重遮挡仍会影响识别

## 当前已知限制

- 首次启动/丢失后的全局重定位耗时仍高。
- 纯色地面、大片水面、草地、弱纹理区域仍有跑偏或丢失风险。
- 资源点密集区、UI 遮挡、数字标记过多时，局部模板跟踪可能会漂。
- 当前识别逻辑仍以 `17173` 地图坐标为主，`biliwiki` 是通过稳定对齐矩阵接入显示坐标；显示层已对齐，但底层识别引擎尚未完全迁移到 `biliwiki` 原生坐标空间。
- `biliwiki` 大图资源是首次使用时自动下载到 `data/rocom_biliwiki/`，不建议直接把这些大体积资源随手提交进仓库。

## 下一步最值得继续做什么

### 1. 继续压缩全局重定位时间

优先级最高。

建议方向：

- 给大地图特征和模板候选做磁盘缓存
- 引入更强的 coarse-to-fine 检索
- 先做低分辨率全图粗搜，再做局部精修
- 必要时参考 `zkjisj/luoke_location` 的 SIFT 锚点缓存思路

### 2. 做“路线跟随模式”

把当前能力从“能用”推进到“更像产品”。

建议做法：

- 正常跟踪时按 `0.5-1s` 稳定刷新
- 丢失时自动切到更积极的重定位策略
- 对路线模式单独展示当前位置与路线进度

### 3. 继续强化真实小地图遮罩

当前已经做了一轮，但还不够。

重点继续看：

- 圆外玩家画面残留
- 中心箭头
- 右上角资源/数字堆叠
- 边缘 UI 装饰
- 特殊场景图标

### 4. 做更系统的性能观测

现在已有零散实测，但还没有真正的 profiling 面板。

建议补：

- 采集耗时
- 预处理耗时
- 全局匹配耗时
- 局部跟踪耗时
- GUI 刷新耗时
- 丢帧数与有效识别率

## 关键文件

- `configs/rocom_17173.yaml`：当前主配置
- `src/preprocess.py`：小地图裁剪、圆区遮罩、干扰抑制
- `src/global_localizer.py`：全局定位核心
- `src/tracker.py`：局部跟踪与状态切换
- `src/pipeline.py`：主识别流程
- `src/async_frame_pipeline.py`：异步采集/识别流水线
- `src/gui.py`：当前主 GUI
- `src/poi_overlay.py`：点位投影、筛选、icon 绘制
- `src/resource_routes.py`：资源路线生成与缓存
- `src/resource_sources.py`：`17173` / `biliwiki` 资源数据源切换
- `src/map_alignment.py`：`17173` 与 `biliwiki` 底图坐标对齐
- `src/utils.py`：配置结构、结果展示与辅助函数
- `tests/test_pipeline.py`：识别流程测试

- `tests/test_resource_routes.py`：资源路线测试

## 运行方式

GUI：

```bash
python main.py --gui
```

命令行单图：

```bash
python main.py --config configs/rocom_17173.yaml --frame /path/to/image.png --save-visualizations
```

命令行持续屏幕采集：

```bash
python main.py --config configs/rocom_17173.yaml --screen-region 1800,260,220,220 --capture-interval-ms 250 --visualize
```
